# PostgreSQL Integration Documentation

## Overview

This project uses PostgreSQL to store semiconductor test simulation results generated by the Python script.

Originally, the Python simulator generated a CSV report. In this sprint, the CSV data was imported into a PostgreSQL database to allow SQL-based validation, metrics analysis, and future dashboard integration.

This simulates a common Application Support Engineering workflow where application-generated data is stored in a database and later reviewed for operational or engineering insights.

## Database Purpose

The PostgreSQL database stores test execution results such as:

* Timestamp
* Device ID
* Test name
* PASS/FAIL status
* Test duration in seconds

This allows the data to be queried using SQL instead of only reviewing flat CSV files.

## Database Created

Database name:

```text
semiconductor_lab
```

Application database user:

```text
lab_user
```

Main table:

```text
test_results
```

## Table Structure

The table was created using the SQL script located at:

```text
sql/create_test_results_table.sql
```

The table stores each test execution as a separate record.

Main columns:

* `id`
* `timestamp`
* `device_id`
* `test_name`
* `status`
* `duration_seconds`
* `created_at`

## CSV Import

The generated CSV report was imported into PostgreSQL using the `\copy` command.

Source file:

```text
reports/test_results.csv
```

Import result:

```text
COPY 25
```

This confirms that 25 records were successfully imported into the database.

## Validation Queries

### Total Records

```sql
SELECT COUNT(*) FROM test_results;
```

Result:

```text
25
```

### PASS/FAIL Summary

```sql
SELECT status, COUNT(*) 
FROM test_results 
GROUP BY status;
```

Result:

```text
FAIL | 3
PASS | 22
```

### Average Test Duration

```sql
SELECT ROUND(AVG(duration_seconds), 3) AS average_duration 
FROM test_results;
```

Result:

```text
0.823 seconds
```
